# Claude Code Release Radar  
要件定義書（v0.1）

---

## 1. 背景・目的

### 1.1 背景

Claude Code は高頻度でアップデートされており、  
手動でのキャッチアップが困難になっている。

特に以下の課題がある：

- リリース頻度が高い
- bugfixなど不要情報も多い
- 手動確認では継続性が低い

---

### 1.2 目的

Claude Code のアップデートのうち、

> **新機能追加および機能拡張のみを抽出し、定期的に通知する仕組みを構築する。**

---

## 2. スコープ

### 2.1 対象

- GitHub上のリリース情報
- CHANGELOG.mdに記載される更新内容

対象リポジトリ：
- GitHub 上の claude-code リポジトリ
https://github.com/anthropics/claude-code

---

### 2.2 対象外

以下は初期スコープ外とする：
- bugfixのみの通知
- 内部改善のみの通知
- Webダッシュボード化
- 複数ユーザー対応
- 商用提供

---

## 3. 機能要件

### 3.1 リリース検知

- 新しいバージョンの公開を検知できること
- 定期実行により確認されること
- リリースタグをトリガーとする案

※実装方式は未確定とする。

---

### 3.2 変更内容取得

- 対象バージョンの変更内容を取得できること
- 対象は CHANGELOG 相当の内容とする

---

### 3.3 フィルタリング

以下の変更のみ抽出する：

- 新機能（Feature）
- 既存機能の拡張（Improvementのうち拡張性を伴うもの）

除外対象：

- Bugfix
- Internal変更
- Refactorのみの変更

※分類方法（ルールベース or LLM判定）は実装時に決定する。

---

### 3.4 要約生成

- 抽出された変更を簡潔に要約すること
- 箇条書き形式で通知可能な形に整形すること

要約粒度：

- 1リリース単位
- 機能単位で分割可能

---

### 3.5 通知

通知先：

- Slack

通知内容：

- バージョン番号
- 新機能一覧
- 機能拡張一覧
- 任意で簡易コメント（将来拡張）

通知頻度：

- 定期実行（毎日想定）

---

## 4. 非機能要件

### 4.1 コスト

- 無料枠内で運用可能であること
- 月額固定費が発生しない構成を前提とする

---

### 4.2 保守性

- 個人運用前提
- 構成はシンプルであること
- 外部依存は最小限

---

### 4.3 拡張性（将来想定）

将来的に以下へ拡張可能な構成であること：

- 重要度スコアリング
- 影響分析の自動生成
- Web UI化（Github Pages）

※初期段階では実装しない。

---

## 5. 制約条件

- 個人用途
- 1日1回の定期実行を想定
- 通知は単一ユーザー向け

---

## 6. 仮決め事項（後工程で確定）

| 項目 | 現時点 |
|------|--------|
| 実行基盤 | GitHub Actions想定 |
| フィルタ方式 | LLMベース想定 |
| 差分検知方法 | リリースタグベース想定 |
| データ保存 | 永続化方法は未決定 |

---

## 7. 成果物

- 自動通知システム
- Slack通知メッセージ
- 将来拡張可能な設計基盤

---

## 8. まとめ

本システムは、

- Slackへの自動通知
- 新機能・拡張のみ抽出
- 定期実行
- 無料前提
- 将来拡張可能

を満たす最小構成の自動キャッチアップ基盤とする。
